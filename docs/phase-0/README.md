# Phase 0: ชุดเตรียมความพร้อม

สถานะ: กำลังดำเนินการ

เอกสารชุดนี้แปลงข้อกำหนดใน [requirements.md](../requirements.md) เป็นข้อมูลและกติกาที่ใช้เริ่ม Phase 1 ได้ โดยยังไม่ถือว่าเป็นข้อมูลจริงจนกว่าผู้ดูแลจะกรอกและอนุมัติ

## สิ่งที่จัดทำแล้ว

| ผลส่งมอบ | ที่อยู่ | สถานะ |
|---|---|---|
| ข้อกำหนดฐานข้อมูล | [database-contract.sql](database-contract.sql) | รออนุมัติก่อนทำ migration |
| กติกาการทำงานและซิงก์ | [operating-rules.md](operating-rules.md) | รออนุมัตินโยบาย |

## ข้อมูลที่ต้องยืนยันก่อนเริ่ม Phase 1

1. ประเภทน้ำแข็งจริง รหัสที่ใช้ และหน่วยนับของแต่ละประเภท
2. รายการร้านจริง: รหัสร้าน ชื่อ อาคาร ชั้น/โซน รูป จำนวนรอบปกติ สถานะจ่ายเงิน เบอร์ติดต่อ และข้อสังเกตในการเข้าร้าน
3. รายชื่อผู้ใช้คนแรก พร้อมบทบาท `courier`, `round_lead` หรือ `admin`
4. ค่ากำหนดหน้างาน: เวลาสูงสุดที่พนักงานแก้ไขรายการของตนได้ และผู้มีสิทธิ์อนุมัติการยกเลิก
5. ราคามาตรฐานของน้ำแข็งแต่ละประเภท และร้านใดมีราคาพิเศษ
6. ร้านใดเก็บเงินรายวัน ร้านใดเป็นเครดิต พร้อมรอบวางบิล จำนวนวันเครดิต และวงเงินเครดิต (ถ้ามี)
8. นิยามรอบสุดท้ายของวัน ผู้มีสิทธิ์ยืนยันสิ้นวัน และวิธีจัดการเมื่อมีการส่งเพิ่มหลังสร้างสรุปยอดแล้ว
9. ขนาดกระดาษ/รุ่นเครื่องพิมพ์ รูปแบบเลขที่เอกสาร และข้อมูลหัวเอกสารที่ต้องแสดง

## เกณฑ์อนุมัติ Phase 0

- ผู้รับผิดชอบหน้างานยืนยันว่าพนักงานค้นหาและเลือกร้านที่ต้องไปส่งเองได้จากบัตรร้านทั้งหมด
- ผู้รับผิดชอบระบบยืนยัน schema รวมยอดตรวจรอบ และกติกาออฟไลน์/ซิงก์ในเอกสารนี้

เมื่อครบทั้งหมด จึงเริ่ม migration และหน้าจัดการข้อมูลของ Phase 1 โดยใช้ schema นี้เป็นจุดตั้งต้น
