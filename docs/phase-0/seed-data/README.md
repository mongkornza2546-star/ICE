# แบบฟอร์มข้อมูลตั้งต้น

กรอกไฟล์แยกตามประเภทข้อมูลเพื่อลดความผิดพลาดในการนำเข้า แล้วนำเข้าตามลำดับนี้:

1. [buildings.csv](buildings.csv)
2. [building-zones.csv](building-zones.csv)
3. [routes.csv](routes.csv)
4. [ice-types.csv](ice-types.csv)
5. [users.csv](users.csv)
6. [shops.csv](shops.csv)
7. [route-shops.csv](route-shops.csv)

## กติกา

- `code` ต้องไม่ซ้ำภายในประเภทข้อมูลเดียวกันและไม่เปลี่ยนหลังเริ่มใช้งาน
- `route_code`, `shop_code`, `building_code` และ `zone_code` ต้องมีอยู่ในไฟล์ต้นทางก่อนนำเข้าไฟล์ที่อ้างถึง
- `zone_code` ต้องไม่ซ้ำภายในตึกเดียวกัน และร้านต้องเลือกโซนย่อยของตึกนั้น ห้ามกรอกชื่อโซนเป็นข้อความอิสระ
- `route-shops.csv` เป็นข้อมูล legacy สำหรับประวัติรอบเก่าเท่านั้น; รอบใหม่ไม่ใช้เส้นทางหรือลำดับร้าน
- `is_active` ของ `route-shops.csv` ใช้ปิดการนำร้านออกจากรอบใหม่โดยไม่ลบประวัติ; ค่าเริ่มต้นคือ `true`
- `image_path` เป็น path ในพื้นที่เก็บไฟล์ ไม่ใช่ URL ชั่วคราว; เว้นว่างได้ระหว่างเตรียมข้อมูล
- `payment_status` ใช้ `unknown`, `paid` หรือ `unpaid`; ค่าเริ่มต้นคือ `unknown`
- `normal_rounds_per_day` ต้องเป็นจำนวนเต็มมากกว่า 0
- ค่าเฉลี่ยและยอดส่งล่าสุดไม่ต้องกรอก เพราะคำนวณจากรายการส่งที่ยังไม่ถูกยกเลิก
