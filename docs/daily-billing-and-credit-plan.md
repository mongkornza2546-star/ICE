# แผนสรุปยอดสิ้นวัน ร้านเครดิต และการพิมพ์เอกสาร

เอกสารนี้ขยาย Phase 3.5 ใน [development-phases.md](development-phases.md) สำหรับการทำงานจริงที่พนักงานสรุปยอดซื้อของแต่ละร้านเมื่อสิ้นวัน และเก็บเงินตอนส่งน้ำแข็งรอบสุดท้าย โดยแยกร้านเครดิตออกจากร้านที่เก็บเงินรายวัน

## Flow หน้างาน

1. พนักงานบันทึกน้ำแข็งที่ส่งในแต่ละรอบตามปกติ ระบบเก็บจำนวน ราคาต่อหน่วย และยอดเงิน ณ เวลาส่ง
2. เมื่อปิดรอบสุดท้าย หัวหน้ากดยืนยันสิ้นวัน ระบบรวมรายการส่งทุกครั้งของวันเป็นสรุปยอดแยกร้าน
3. ร้าน `เก็บเงินรายวัน` ปรากฏในรายการเก็บเงินรอบสุดท้าย พนักงานพิมพ์ใบสรุปยอด/ใบแจ้งยอดและนำไปให้ร้านตรวจ
4. เมื่อรับเงินจริง พนักงานบันทึกยอดรับและวิธีชำระ ระบบจึงออกใบเสร็จรับเงิน หากรับไม่ครบ ส่วนที่เหลือกลายเป็นยอดค้าง
5. ร้าน `เครดิต` ไม่ต้องเก็บเงินในรอบสุดท้าย ยอดของวันถูกสะสมเข้าลูกหนี้ตามรอบวางบิลและวันครบกำหนด
6. เมื่อร้านเครดิตชำระ แอดมินบันทึกรับเงินและเลือกว่าจะตัดเอกสารใด ระบบออกใบเสร็จตามยอดรับจริง

## ข้อมูลที่ต้องเพิ่ม

- เงื่อนไขร้าน: `payment_term`, รอบวางบิล, จำนวนวันเครดิต, วงเงินเครดิต และสถานะอนุญาตเครดิต
- ราคา: ราคามาตรฐานและราคาพิเศษรายร้าน/ชนิดน้ำแข็ง พร้อมช่วงวันที่มีผล
- รายการส่ง: ราคาต่อหน่วยและยอดเงิน snapshot
- เอกสาร: เลขที่ ประเภท วันที่ ร้าน รายการต้นทาง ยอดรวม วันครบกำหนด สถานะ และเอกสารทดแทน
- การรับชำระ: จำนวน วิธีชำระ เวลา ผู้รับเงิน เลขอ้างอิง และ idempotency key
- การจัดสรรรับชำระ: เงินรับหนึ่งรายการตัดยอดเอกสารใดเป็นจำนวนเท่าใด
- การพิมพ์: เวลา ผู้พิมพ์ จำนวนครั้ง และสถานะสำเนา โดยไม่ใช้การพิมพ์เป็นหลักฐานว่ารับเงินแล้ว

## หน้าจอที่ต้องมี

- หน้าตั้งราคาและเงื่อนไขชำระของร้าน
- หน้ายืนยันสิ้นวันและตรวจสรุปยอดก่อนออกเอกสาร
- หน้ารายการเก็บเงินรอบสุดท้ายสำหรับพนักงาน
- หน้าบันทึกรับเงินและพิมพ์ใบเสร็จ
- หน้าลูกหนี้เครดิต แยกยังไม่ครบกำหนด ครบกำหนด และเกินกำหนด
- หน้าค้นหาเอกสาร พิมพ์ซ้ำ ยกเลิก และดู audit log

## กรณีที่ต้องทดสอบ

- ร้านเดียวซื้อหลายรอบและหลายชนิดน้ำแข็งในวันเดียว
- เปลี่ยนราคาหลังส่งแล้ว แต่ยอดเอกสารเดิมไม่เปลี่ยน
- กดยืนยันสิ้นวันหรือพิมพ์ซ้ำหลายครั้งโดยไม่เกิดเอกสารซ้ำ
- รับเงินเต็มจำนวน บางส่วน เกินยอด และรับหนึ่งครั้งเพื่อตัดหลายเอกสาร
- ยกเลิกรายการส่งก่อนและหลังออกเอกสาร โดยใช้เอกสารปรับปรุงแทนการเขียนทับยอด
- มีการส่งเพิ่มหลังยืนยันสิ้นวัน ระบบต้องแจ้งให้สร้างเอกสารเพิ่มเติมหรือเอกสารทดแทน ห้ามแก้ใบเดิมเงียบ ๆ
- ออฟไลน์ระหว่างรับเงินแล้ว retry โดยไม่เกิดรายการรับเงินหรือใบเสร็จซ้ำ
- ร้านเครดิตครบกำหนด/เกินกำหนด และร้านเครดิตถูกระงับชั่วคราวเมื่อเกินนโยบายที่กำหนด

## เรื่องที่ต้องยืนยันก่อนพัฒนา

1. ร้านเครดิตวางบิลรายวัน รายสัปดาห์ รายครึ่งเดือน หรือรายเดือน และให้เครดิตกี่วัน
2. ราคาเป็นราคาเดียวทุกร้านหรือมีราคาพิเศษ และมีส่วนลด/ภาษีหรือไม่
3. ใช้เครื่องพิมพ์ Bluetooth/thermal รุ่นใด และกระดาษ 58 มม., 80 มม. หรือ A4
4. ข้อมูลบนหัวเอกสารและรูปแบบเลขที่เอกสารที่ต้องการ รวมถึงข้อกำหนดทางบัญชี/ภาษี
5. ผู้ใดมีสิทธิ์ยืนยันสิ้นวัน รับเงินสด ยกเลิกรับเงิน และอนุมัติเครดิต
